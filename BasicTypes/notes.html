Facilities

  • Parse text to C# data structure.
  • Support C# data facilities, such as Equality, Comparison, Serialization, TypeConversion, Mapping, etc
  • Serialize C# data structure to a lossy, human readable string.
  • Serialize to alternative formats, namely, English Gloss and Structure Annotated

To and From Strings

The basic types can be converted to human readable format using ToString(). Be careful to not accidentally call the object.ToString() base method.

The basic types can be converted from human readable format using two methods: Parse(value), TryParse(value) or using the XXXTypeConverter.

Serialization

Types support binary, XML, and JSON serialization. This differs from Parse/ToString because it isn't (very) lossy and the target consumer is a computer.

To and From Other DataType

Except for numbers, there isn't a natural way to convert among types, since they are at best similar. So for these type conversion is done with AutoMapper

Equality

All types are Classes, so are left to use their out of the box equality.

Equality by value is supported with a XXXByValue class.

Collection equality is an important concept-- collections can be the same or different base on:

  1. Same but different order, e.g. strings of modifiers
  2. Exact and ordered, e.g. strings of verb phrases (temporal interpretation)
  3. Close enough (most match)

Facts

Facts have scope. Within a scope there are bound facts, which have something that is unique to the word. That would be a noun with a proper modifier.

Scopes can be linked via matching proper modifiers
